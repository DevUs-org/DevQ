DevQ Phase 4 sanity test plan
Run `python main.py` from the project root (3 devices: d0 random devq 7q, d1 FakeNairobiV2, d2 FakeLagosV2).
Config files are referenced by name; all live in config/config_examples/. A block's stated config
stays in effect for every following block until a block states a new one.
NOTES: d0's error map is random per launch — deterministic checks pin --exec=d1,d2 or exclude d0.
Aer counts are unseeded — check mappings/states/reasons, not counts. Job IDs assume this exact
order in a fresh session. d1/d2 calibration values assume qiskit-ibm-runtime 0.45.1 — verify
Block 1's qerrors output first; if those match, everything downstream should too.

Block format:  Block N ~ Instructions ~ Commands ~ Expectations ~

qubit_error_weight 0.1 / edge_error_weight 0.9 source User (global) — the raw 1/9 is

pair). qconfig d1 shows 0 / 1 source User (d1) (per-device override of the common key).

the RATIO equals the 0.1/0.9 default. Bonus: zero_weights.config.json prints a
